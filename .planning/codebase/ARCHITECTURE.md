# 架构

**分析日期：** 2026-04-29

## 总结

项目采用静态单页架构，分为三层：结构层（HTML）、表现层（CSS）、行为层（JS）。浏览器直接加载，无服务端应用层。

## 分层说明

### 结构层
- 文件：`index.html`
- 职责：定义导航、区块、文案容器、资源引用。

### 表现层
- 文件：`styles.css`
- 职责：主题变量、布局、断点与组件样式。

### 行为层
- 文件：`main.js`
- 职责：菜单开关、锚点点击后的状态收敛。

## 关键数据流

1. 点击移动端菜单按钮。
2. JS 切换 `#nav.open` 与 `aria-expanded`。
3. CSS 根据 `open` 状态控制可见性。
4. 点击锚点后菜单自动关闭并执行原生滚动。

## 架构特点

- 简单直接，交付效率高。
- 无构建链，维护门槛低。
- 结构与脚本耦合较高，需通过约束降低回归风险。
