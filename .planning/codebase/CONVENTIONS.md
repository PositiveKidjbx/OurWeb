# 编码约定

**分析日期：** 2026-04-29

## 现有约定

- 命名以语义化 class/id 为主。
- 样式使用 CSS 变量管理颜色与基础设计 token。
- JS 逻辑偏事件驱动，结构简洁。
- 资源命名遵循小写与中划线风格（按 PRD 规范）。

## 一致性观察

- HTML/CSS/JS 命名整体一致，易读性较好。
- 交互逻辑短小，但对 DOM 结构依赖强。
- 缺少明确的 lint/format 规范与自动检查流程。

## 建议

- 增加轻量级代码规范文档（命名、注释、目录边界）。
- 为关键交互函数补充最少必要注释。
- 若项目扩展，考虑引入格式化与静态检查工具。
